About Me

Hi! My name is Sheldon Chean and I'm a software developer based in Auckland, New Zealand.

I am currently a student at the University of Auckland, taking a Bachelor of Science in Computer Science and Information Systems. I enjoy the relationship between these two majors as I believe the business skills and different teachings provides me a unique view on software development, mainly in producing software for clients.

The everchanging world of web development was intimidating to step into at first, but discovering the new frameworks and keeping up with the bleeding edge has been exciting. Despite coming from a more fundamental education in software development, I am always willing to learn the new ideas and techniques that lie in web development.

What are my interests outside of programming?

In my free time I am an avid competitor in competitive fighting games. Since joining the New Zealand scene I have made deep connections with people from all walks of life, experienced what goes into hosting large events, and greatly improved my problem solving skills. I also enjoy consuming all sorts of media, specifically literature and animation. Outside of these, I have a keen interest in fashion, graphic design, writing and learning Japanese.

Type Poetry

Type Poetry


Type Poetry is a web app that allows users to 'retype the greats' of poetry. Users can type along with almost 3000 classic poems, or generate random poems using this collection. This was designed for students looking for a unique way to study poems, and for secondary language English learners to experience a new type of text.

This was built using the MEAN stack and my focus was incorporating new technologies and ideas into my workflow. These included task runners (Gulp), CSS preprocessors (SASS) and deployment (DigitalOcean). I created Gulp tasks for serving dev/production builds, minifying code, injecting dependencies and vetting code. This significantly helped improve my workflow and allowed the deployment of a much more optimized application.

Kisetsu

Kisetsu (Still in Development)


Currently there is an error in the AniList API causing the app to not function.

Kisetsu is a web app that allows users to follow the current anime season and vote after each episode airs on how much they enjoyed it. I created this as I found users in the community would commonly debate over what is most popular during the airing season, so this would be an interesting and easy way to show what is trending.

This was my second project using the MEAN stack, this time following a rapid application development philosophy. I took inspiration from Simon Holmes' book, 'Getting MEAN', and followed best practices as much as possible. It uses a wrapper API to interact with a third-party API and the main database where ratings are stored. It makes use of cookies and weighted algorithms with smoothing applied, similar to Amazon's system. This is still in development and I am looking at adding many new features, including more build automation and Socket.io integration for viewing of live votes.

gamestack

gamestack


gamestack is a website where users can register accounts and list the video games they own in their collection. Users are able to add and remove games from their 'stack', and also view the 'stacks' of other users. It features safe and persistent sessions, responsive design, and descriptive error handling.

This was my first approach at Node.js, Express.js and Mongoose. I learned a lot of important lessons when developing this, especially interacting with external APIs and the downsides of not using a front-end framework.

Popup Gazebo Website

Popup Gazebo Website


This was a website created for a client who receives thousands of unique visitors each month. Instead of using Wordpress as a traditional solution, I opted for a more lightweight CMS in CouchCMS to define editable regions to make it simpler for the client.
